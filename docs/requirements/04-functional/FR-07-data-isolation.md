## 概要

当面はローカル単一ユーザーのためデータ分離は不要。マルチユーザー化（認証導入）時に、ユーザー本人の記録のみ閲覧・編集できるよう設計する。

> 旧方針（Firestore セキュリティルールによる分離）は、技術構成を Go + MySQL に変更したため撤回（[12-open-issues.md](../12-open-issues.md) Q-07）。

## 将来導入する場合の方針（参考）

- 各レコードに userId（所有者）を持たせ、API 層で認証ユーザーの userId に限定して読み書きする。
